# CPU Usage Track

一款轻量级 CPU 实时监控桌面应用，基于 PyQt5 深色主题界面，CPU 占用极低（< 1%）。

![Python](https://img.shields.io/badge/Python-3.10+-blue) ![PyQt5](https://img.shields.io/badge/PyQt5-5.15-green) ![License](https://img.shields.io/badge/License-MIT-yellow)

---

## 功能特性

- 🖥️ **实时 CPU 曲线** — pyqtgraph 绘制，最多保留 300 个数据点（约 5 分钟 @1s 采样）
- 🔔 **智能告警** — CPU 超阈值时自动记录 Top 10 进程，生成 Markdown 日志
- 📊 **数据导出** — 一键保存监控数据为 CSV 文件
- ⚡ **极低开销** — 分级采集策略，常规路径 CPU 占用 ≈ 0.05%–0.15%
- 🎨 **深色主题** — GitHub Dark 风格界面，长时间使用不刺眼
- ⏸️ **暂停/恢复** — 随时中断和继续监控，运行时长精确扣除暂停时间

## 使用流程

### 1. 开始监控

点击 **▶ 开始** 按钮，应用开始采集 CPU 数据。曲线图实时更新，状态栏显示当前 CPU 使用率和运行时长。

### 2. 调整参数（运行中也可以改）

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| **采样间隔** | 1–60 秒 | 1 秒 | 数据采集频率。增大间隔可进一步降低 CPU 开销 |
| **告警阈值** | 1–100% | 90% | CPU 超过此值时触发告警。阈值线会在曲线图上实时移动 |

修改参数后立即生效（正在运行时也会动态更新工作线程配置）。

### 3. 告警触发

当 CPU 使用率 **≥ 告警阈值** 时：

1. 自动抓取当前 **CPU 占用率 Top 10** 进程
2. 生成一份 Markdown 格式的告警日志，保存到 `logs/` 目录
3. 告警列表新增一条红色记录
4. 告警次数 +1

**防刷机制**：同一告警事件最少间隔 30 秒（`log_cooldown`），不会疯狂刷日志。

#### 告警日志示例

```markdown
# CPU 告警日志

- **时间**: 2026-04-21 14:32:05
- **CPU 使用率**: 95.2%
- **告警阈值**: 已超过设定阈值

## CPU 占用率 Top 10 进程

| 排名 | 进程名 | PID | CPU 占用率 |
|------|--------|-----|-----------|
| 1 | chrome.exe | 1234 | 45.2% |
| 2 | devenv.exe | 5678 | 22.1% |
| ... | ... | ... | ... |
```

日志文件命名格式：`cpu_alert_YYYYMMDD_HHMMSS.md`

### 4. 暂停 / 恢复

- 点击 **⏸ 中断** → 暂停数据采集，曲线冻结，运行时长停止计时
- 暂停后按钮变为 **▶ 恢复** → 点击恢复采集，运行时长继续（精确扣除暂停时间）

### 5. 保存数据

点击 **💾 保存**，选择保存路径，导出为 CSV 文件。

CSV 文件格式：

| 列名 | 类型 | 示例 |
|------|------|------|
| `timestamp` | Unix 时间戳 | 1745220725.123 |
| `datetime` | 可读时间 | 2026-04-21 14:32:05 |
| `cpu_percent` | CPU 百分比 | 95.2 |

### 6. 停止监控

点击 **⏹ 停止**，结束监控。停止后仍可保存已采集的数据。再次点击 **▶ 开始** 会清空旧数据重新开始。

## 架构设计

```
主线程（PyQt5 事件循环）          工作线程（QThread）
┌──────────────────┐           ┌──────────────────┐
│  MainWindow      │           │  CPUMonitorThread │
│  ├── 曲线更新     │  ◄─────── │  ├── cpu_percent  │  常规路径：每秒 < 0.1ms
│  ├── 状态栏更新   │  Signal   │  │                │
│  ├── 告警列表     │  ◄─────── │  └── process_iter │  告警路径：仅超阈值+冷却期
│  └── 日志生成     │  Signal   │                  │
└──────────────────┘           └──────────────────┘
```

- **常规路径**：仅调用 `psutil.cpu_percent(interval=None)`，开销 < 0.1ms
- **告警路径**：超阈值且冷却期已过时才遍历进程列表，避免频繁高开销操作
- **线程通信**：Signal/Slot 机制，数据从工作线程安全传递到主线程更新 UI

## 常见问题

### Q: 启动后第一个数据点为什么偏低？

`psutil.cpu_percent(interval=None)` 需要两次调用之间的时间差来计算。启动时有一次预热调用，但预热和首次采集间隔极短，所以第一个点可能接近 0。这是正常现象，不影响后续数据准确性。

### Q: 应用自身 CPU 占用是多少？

常规路径下约 0.05%–0.15%，远低于 1%。即使触发告警（遍历进程列表），由于有 30 秒冷却期限制，对整体开销影响也很小。

### Q: 支持多核 CPU 显示吗？

当前显示的是**整体 CPU 使用率**（所有核心平均值），不是单核。如需单核数据，可以扩展 `monitor.py` 中的 `psutil.cpu_percent(percpu=True)` 参数。

### Q: 告警日志太多怎么办？

日志保存在 `logs/` 目录，可手动清理旧文件。应用内告警列表最多显示最近 20 条。
